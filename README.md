# Словникові картки — PWA для Microsoft Store

Готовий офлайн застосунок (чистий HTML/CSS/JS, без збірки). 813 слів і фраз
у 39 темах, зелений інтерфейс, переклад UA/RU, прогрес зберігається на
пристрої. Нижче — як перевірити його локально і як перетворити на пакет
для Microsoft Store.

## Коротко: 5 кроків до публікації

1. Залити цю папку на GitHub Pages (безкоштовно, HTTPS) — розділ 2.
2. Відкрити pwabuilder.com, вставити посилання на сайт — розділ 3.
3. Натиснути "Package for stores" → Windows → завантажити `.msix`.
4. Зареєструватись у Microsoft Partner Center (одноразово) — розділ 4.
5. Створити заявку на застосунок, завантажити `.msix`, додати скріншоти
   й опис, відправити на перевірку.

## 1. Перевірити локально

Service worker не працює через `file://`, тож потрібен локальний сервер:

```bash
cd flashcards-pwa
python3 -m http.server 8080
```

Відкрити http://localhost:8080 у браузері. Перевірити:
- усі 12 розділів відкриваються, картки перевертаються;
- перемикач UA/RU працює і зберігається після перезавантаження сторінки;
- прогрес "Знаю / Ще вчу" зберігається після перезавантаження (localStorage);
- у DevTools → Application → Service Workers видно, що sw.js зареєстрований.

## 2. Розмістити онлайн (потрібно для PWABuilder)

Найпростіше — залити цю папку на будь-який безкоштовний хостинг для статичних
сайтів: GitHub Pages, Netlify, Vercel, Cloudflare Pages. Головне — сайт має
відкриватись по HTTPS, інакше PWABuilder не згенерує пакет.

Приклад для GitHub Pages:
1. Створити репозиторій, залити вміст цієї папки.
2. Settings → Pages → Deploy from branch → main / (root).
3. Отримаєте посилання типу `https://username.github.io/flashcards-pwa/`.

## 3. Згенерувати пакет для Microsoft Store через PWABuilder

1. Відкрити https://www.pwabuilder.com
2. Вставити URL вашого сайту → "Start".
3. PWABuilder перевірить manifest і service worker (обидва вже готові в
   цьому проєкті) — має показати зелені галочки.
4. Натиснути "Package for stores" → обрати **Windows**.
5. Вказати Package ID / Publisher (їх видає Microsoft Partner Center —
   див. пункт 4) або залишити тестові значення для першої перевірки.
6. Завантажити готовий `.msix` / `.msixbundle`.

## 4. Publisher-акаунт і сама публікація

Для реальної публікації в Store потрібен:
- акаунт розробника в Microsoft Partner Center
  (https://partner.microsoft.com/dashboard) — одноразова реєстрація
  (є безкоштовний варіант для індивідуальних розробників);
- у Partner Center створити нову заявку на застосунок, вказати назву,
  завантажити `.msix`, додати скріншоти (мінімум 1, рекомендовано 3–4),
  опис, вікові обмеження (тут підійде "Для всіх"), категорію "Освіта".

Скріншоти можна зробити просто відкривши сайт у вікні Windows-розміру
1366×768 або 1920×1080 і зробивши знімок головного екрана та екрана картки.

## 5. Що можна покращити перед публікацією

- Додати власний домен/хостинг замість тимчасового (GitHub Pages підходить
  і для постійного використання).
- За бажання — прибрати залежність від Google Fonts (зараз шрифти
  підвантажуються з мережі; офлайн without них використовуються системні
  шрифти — застосунок працює, просто без фірмового шрифту Fraunces).
- Розширити базу слів (зараз 813 слів у 39 розділах, продовжуємо додавати
  партіями до ~1500-2000).
- Додати кілька скріншотів у `icons/` для сторінки в Store (не обов'язково
  для роботи самого застосунку).

## Структура проєкту

```
flashcards-pwa/
├── index.html          — головна сторінка
├── manifest.json        — PWA-маніфест (назва, іконки, кольори)
├── sw.js                 — service worker (офлайн-кеш)
├── css/styles.css
├── js/data.js            — усі слова/фрази за категоріями
├── js/app.js             — логіка застосунку
└── icons/                — іконки 48/144/192/512 + maskable
```
